# TestFlowAI — Work Log (2026-06-05)

A complete record of the changes made today, turning the app into a single AI‑native, agent‑driven QA product. Frontend runs on **:3000** (Vite), backend on **:3001** (Express) — see `dev:frontend` / `dev:backend` scripts. Backend changes require a backend restart; frontend hot‑reloads.

---

## 1. Agent Console as the home screen
- New `src/pages/AgentConsole.tsx` is the landing route (`/`). The classic Dashboard moved to `/dashboard`. The **full sidebar is kept** (client wanted both).
- One conversational surface: type/speak a request → the controller builds a plan → you approve → it executes. Decisions land in the AI Inbox.
- Starter prompt chips, voice input, a capability strip ("everything the agent can do"), and a wider layout.

## 2. Merged "AI Agent" (Test Studio) into the Console
- Removed the separate **AI Agent** sidebar entry. The old page is kept only at `/studio` (unlinked fallback).
- The console now **drives the deep generation pipeline inline** (`src/components/DeepRunResult.tsx`): inspect app → generate cases → Playwright scripts → evidence, with live pipeline progress.
- Routing in `AgentConsole`: a request with a URL / generation keywords → deep pipeline; operational requests (plans/suites/runs/defects/reports/folders) → controller plan→approve→execute; questions → chat.

## 3. Inline case editor in the Console
`DeepRunResult` lets the user do all the "drudge" work in chat: edit case (title/desc/priority/tags), add a case, add/remove/edit steps, **AI‑expand steps**, **rework with feedback**, plus the review gate (**Save all** / **Continue → scripts & evidence**). Deep runs now use `flowMode: 'review_cases'` so they pause for review.

## 4. Website Credentials (Settings)
- **Fixed a real persistence bug**: `website_users` had no `updated_at`, so PG upserts failed silently and credentials were wiped on restart. Added idempotent `ALTER TABLE` migrations (`updated_at`, `page_name`, `page_url`).
- Reworked the UI into a **flat "Website Credentials" table** (Website Name · URL · Username · Password · "Use for Playwright" · delete), with **Add Website** and inline auto‑save. Replaced the nested Website→Logins panel per request.

## 5. Chat persistence
- New `chat_conversations` table + `server/features/chat/routes.ts` (GET/PUT/list/delete). The console autosaves the conversation, restores it on refresh, and has a **History** dropdown + **New** button.

## 6. Codebase change analysis (git → tests)
- `server/features/git-agent/analysisService.ts` + `/api/git-agent/analyze` + `/apply`. Reads the **real diff**, classifies each change (`ui` / `functional` / `business-logic` / **`api`** / **`db-schema`** / `config`), **reconciles against existing cases/scripts**, and proposes only the gap tests.
- Console renders it via `src/components/CodeChangeReview.tsx` (change list, "your existing tests already cover this" vs gaps, one‑click create). Triggered by phrases like "check the recent code changes and test them".

## 7. Agent consolidation (14 → 7 roles)
- Consolidated to: **Chat Assistant, Case Writer, Test Planner, Suite & Folder Organizer, Playwright Coder, Application Inspector, Defect & Report Analyst**. Legacy agents are **aliased** (`AGENT_ALIASES` + `canonicalAgent()`), so every existing call still works. The System Prompts UI shows only the 7 (`CANONICAL_AGENTS`). Added a locked‑down internal `searchAgent`.

## 8. Real Playwright execution
- `server/features/playwright/executionService.ts` + `/api/playwright/run`: writes generated specs to a temp project, runs `npx playwright test --reporter=json`, parses **true pass/fail + per‑test errors**. Installed `@playwright/test`.
- Console Scripts tab: **"Run all scripts"** button + an honest "Drafts — not executed yet" label, then per‑test ✓/✗ results. Verified end‑to‑end (a real `page.goto` test passed, a bad assertion failed with its real error).

## 9. AI providers
- Swapped the hand‑rolled `fetch` providers to the **official SDKs**: `openai` and `@anthropic-ai/sdk` (Gemini stays on the `ai` SDK). Typed‑error mapping; Opus‑4.7/4.8 `temperature` guard. Model stays user‑selected.
- **Latest models only** (`server/ai/providers/types.ts`): OpenAI default `gpt-5.4-mini` (alts `gpt-5.5`/`gpt-5.4`/`gpt-5.4-nano`), Anthropic default `claude-opus-4-8` (alts `claude-sonnet-4-6`/`claude-haiku-4-5`). Updated `.env.example` with `OPENAI_API_KEY`/`ANTHROPIC_API_KEY`/`CRED_ENC_KEY`.

## 10. Cleaner, smarter chat
- `explainIntent` produces well‑structured **plain text with no markdown/special characters/emojis** (prompt + `sanitizeAnswer` server‑side, `cleanChat` client‑side).
- **Ambiguity gate**: low‑confidence interpretations show a "Just so I get it right — did you mean X, or something else?" card with Yes / No before acting.

## 11. Memory / history‑aware agents
- The orchestrator can be fed **workspace context** (recent cases/suites/scripts/plans/runs/defects with ids + timestamps) and **recent conversation**, so it answers "the cases you made 2 days ago" with real ids and resolves "tweak those and rerun" to concrete ids.
- **Conditional**: context is only fetched when the request references past work (`needsHistory` heuristic) — plain requests stay fast.
- Added a shared **`GROUNDING_POLICY`** composed into every agent: ground in inputs (no fabrication), consume/produce across agents (handoffs), stay in lane, human‑in‑the‑loop (never claim a test ran unless it did), credentials referenced never emitted, concrete over verbose.

## 12. `@ai` smart search (all list pages)
- Type `@ai <natural language>` in the search bar of **Test Cases, Suites, Plans, Runs, Defects, Reports** → a **search‑only** agent filters the **current page's data**. Reusable `src/lib/useAiSearch.ts`.
- Guardrails: `searchAgent` prompt forbids anything but selecting ids; backend caps/trims items and **intersects returned ids with the ids actually on the page** (`server/features/search/routes.ts`), so it can't act, answer, or return anything outside the current data.

## 13. Streaming console responses
- Added `generateTextStream` to all three providers, `AgentOrchestrator.streamText` (keeps the guardrail pipeline), `/api/controller/explain/stream`, and the console reads the body incrementally → chat answers **stream token‑by‑token** (falls back to non‑streaming).

## 14. Downloadable agent‑run report
- `DeepRunResult` has a **"Report ▾"** menu → **PDF / HTML / Excel(CSV)**, built from the run's cases + steps + Playwright pass/fail + evidence screenshots.

## 15. Test any saved site by name
- Mention a stored website name (e.g. "test the **admin** login, does it work") → the console resolves its **URL + saved credentials** and runs the deep pipeline focused on the request. Works for **any QA use case** on the site (test/check/verify/generate/explore), not just login; plain info questions stay in chat. (`findWebsiteInText`, `siteActionable`, passes `websiteId` to `/api/agent/start`.)

## 16. File System control from the Console
- New intents **`organize_repository`** (AI proposes a folder structure and files artifacts) and **`move_to_folder`** (move specific cases/suites/scripts into a folder) — real `Folders.upsert` + `folderId` updates, routed through plan→approve→execute, with File System drill‑in links.

## 17. Autonomy moved to chat
- Removed the **Autonomy** tab from Settings. Set it from the console chat ("set autonomy to manual / review / autonomous") via the existing `set_autonomy` intent (now using the app's real values), with a small tip shown in the console empty state.

## 18. Layout / UX
- Sidebar: **narrower** (w‑56), nav grouped into collapsible sections (Overview / Test Management / Quality / Automation). Settings sections **wider** (max‑w‑7xl).
- **Collapsing the sidebar widens the screens app‑wide**: `<main data-sidebar>` + a single CSS rule `main[data-sidebar="closed"] .max-w-7xl { max-width: none; }`.
- Subsuites in Test Suites (parent picker, "Add subsuite" per row, "↳ Subsuite of …" indicator).
